发表于 2020-04-25

前言

在前后端分离的项目上，目前主流的前端部署方案是，前端单独打包一个docker镜像，用nginx来做http访问和后端服务代理，这样既可以解决前后端分离的跨域问题，也可以隐藏真实的后端地址；

修复

经过查阅，发现envsubst可以指定需要替换的变量

多个变量使用空格隔开即可 envsubst '${BACKEND_URL} ${OTHER_VAR1} ${OTHER_VAR2}' ...
